# /reprioritise
Re-sequence the backlog after something changed.
## Steps
1. **Restate scope** — Confirm what's in and out using `scoping-framework` skill.
2. **Re-score** — Apply the model to all live work using `prioritisation-framework` skill.
3. **Re-check capacity** — See what now fits using `capacity-planning` skill.
4. **Re-trace dependencies** — Update the critical path using `dependency-map` skill.
## Output
An updated, justified priority order with the trade-offs made explicit.
Consider following up with `/report-status` to communicate the change.
